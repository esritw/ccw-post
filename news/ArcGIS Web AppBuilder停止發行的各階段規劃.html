ArcGIS Web AppBuilder停止發行的各階段規劃

27 二月, 2023阿牛哥

ArcGIS Web App Builder即將停止發行，我們想與您分享停止發行時程表和詳細資訊。雖然您已建立的 ArcGIS Web AppBuilder 應用程式仍將繼續運作，原廠仍建議您將應用程式轉移至 ArcGIS Experience Builder，以使用 ArcGIS 的新功能並跟上瀏覽器技術的變化。以下是時程表和詳細資訊：

開發者版本

開發者版本將於 2024 年 7 月停止發行，屆時 ArcGIS API for JavaScript 3.x 版本將一起停止發行。

ArcGIS Enterprise 中的 ArcGIS Web AppBuilder

自即日起，ArcGIS Enterprise 中的 ArcGIS Web AppBuilder 將被視為過時。ArcGIS Web AppBuilder 將持續作為 ArcGIS Enterprise 的一部分，直到 2025 年上半年停止發行，在這之後的新版ArcGIS Enterprise，將不再包含ArcGIS Web AppBuilder。

現行版本的 ArcGIS Enterprise ，將根據 ArcGIS Enterprise 產品生命週期繼續獲得支援。

ArcGIS Online 中的 ArcGIS Web AppBuilder

ArcGIS Online 中的 ArcGIS Web AppBuilder 將於 2025 年第四季度停止發行。 

自 2014 年首次發行以來，我們看到很多ArcGIS Web AppBuilder 使用者建立出令人驚豔的網路應用程式，用以解決企業組織和社會大眾所面臨的挑戰性問題。ArcGIS Web AppBuilder 使用 ArcGIS API for JavaScript（3.x）開發，該 API 將在 2024 年 7 月停止發行。隨著 ArcGIS Web AppBuilder 的底層技術停止開發，ArcGIS Online、ArcGIS Enterprise 和 ArcGIS Developer 訂閱中的 ArcGIS Web AppBuilder 也將停止發行。

在過去幾年中，原廠的網路技術不斷演進和現代化。ArcGIS Experience Builder是基於原廠最新的網路地圖技術ArcGIS Maps SDK for JavaScript開發，它具有最佳化的性能，並提供遠遠超過 ArcGIS Web AppBuilder 所提供的功能與套件，例如「行動裝置優化」、「2D 和 3D 同時存在一個應用程式中」、「靈活地版面設計」、「流程最佳化」和「與其他 ArcGIS 應用程式的整合」。ArcGIS Experience Builder 已經具備了 ArcGIS Web AppBuilder 的大部分功能，因此我們強烈建議您將 Web AppBuilder 應用程式轉移至 Experience Builder。由於瀏覽器更新可能會引入功能或安全問題，原廠將不再解決這些問題。因此，請盡快轉移您的應用程式。

請參閱 Web AppBuilder 和 Experience Builder 之間的功能矩陣以取得更詳細的說明。

常見問題

停止發行後我的應用程式還會繼續運作嗎？

開發者版本：

會。不過將不再進行Bug修復或增強功能。瀏覽器的更新可能會引入功能或安全問題，原廠將不予處理。此外，原廠的技術支援也不再提供。

ArcGIS Enterprise版：

會。ArcGIS Web AppBuilder仍然會得到充分的支援，並將根據ArcGIS Enterprise產品生命週期繼續得到維護。不過它只會接收關鍵性的錯誤修復，例如安全更新，否則將處於維護模式，並沒有計畫進行功能增強。

ArcGIS Online版：

會。您所建立的現有ArcGIS Web AppBuilder應用程式將繼續運作。

在停止發行後，您將無法用它「建立新應用程式」或「編輯現有應用程式」，也不再進行Bug修復或增強功能。瀏覽器的更新可能會引入功能或安全問題，原廠將不予以處理，原廠的技術支援也不再提供。

請參閱產品生命週期支援政策。

我應該採取什麼行動來減緩停止發行的影響？

ArcGIS Web AppBuilder 應用程式可以轉移到 ArcGIS Experience Builder。對於在 ArcGIS Online 和 ArcGIS Enterprise 上的 ArcGIS Web AppBuilder，請觀看這個轉移影片。對於使用開發版建立的客製化應用程式，包括客製化元件和佈景主題，請參閱此說明文件以了解在 ArcGIS Experience Builder 的開發者版中重建元件和佈景主題的技巧、技術和最佳實踐建議。

請注意，某些 ArcGIS Web AppBuilder 元件將無法直接轉移到 ArcGIS Experience Builder，因為它們的功能可以在其他等效的 Experience Builder 元件中找到。在某些情況下，ArcGIS Web AppBuilder 的部分元件功能，現在已經被視為ArcGIS 的基礎功能，例如圖徵圖層的叢集(Clustering on feature layer)，因此可以直接在 Experience Builder 中使用而無需特殊元件。在少數情況下，原廠決定將包含在 ArcGIS Web AppBuilder 中的特殊功能轉移到ArcGIS Instant App，這些應用程式可提供使用者設定和使用。這些詳細資訊可以在 Web AppBuilder 和 Experience Builder 之間的功能矩陣中找到。

此外，您也可以考慮將 ArcGIS Web AppBuilder 應用程式轉移到其他 ArcGIS 應用程式，例如ArcGIS Instant Apps、ArcGIS 儀表板和 ArcGIS 故事地圖。

我能在 ArcGIS Enterprise 中新增客製化元件嗎？

從ArcGIS Enterprise 11.0 開始，您可以像在 ArcGIS Web AppBuilder 中那樣，新增 Experience Builder客製化元件到 ArcGIS Enterprise上。我們強烈建議您將 ArcGIS Web AppBuilder 的客製化元件轉移到 ArcGIS Experience Builder。新增Web AppBuilder客製化元件到ArcGIS Enterprise的功能已被棄用，僅能在不受支援的ArcGIS Web AppBuilder 開發者版本中使用。

停止發行後，我還能繼續下載開發者版本嗎？

停止發行，您仍然可以從ArcGIS Developers網站下載現行版本的 ArcGIS Web AppBuilder 開發者版本。ArcGIS Developers 網站上的開發者版本頁面將更新，以反映已經停止發行的狀態。

轉移作業參考資源

ArcGIS Experience Builder 提供一種新的方式，可以建立現代化的以地圖為核心的應用程式，或是非以地圖為核心的應用程式和網頁。由於它的架構與 ArcGIS Web AppBuilder 不同，因此轉移意味著應用程式需要重新設定或重新撰寫。我們知道您可能會面臨一些挑戰，因此我們分享停止發行計畫以便幫助您做出轉移策略的決定，並分享可用的技術參考資源，讓您可以開始轉換。
